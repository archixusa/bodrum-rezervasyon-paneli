# Tamamlanma Runbook'u

Kullanıcı 3 dış servisi (Supabase, Telegram bot, Resend) açtıkça aşağıdaki komutlar sırayla çalıştırılır.

## Senin vereceğin değerler (bana yaz)

| Değer | Nereden |
|---|---|
| `SUPABASE_PROJECT_REF` | Supabase dashboard URL'inden (`abcdef`) |
| `SUPABASE_URL` | Project Settings → API → Project URL |
| `SUPABASE_ANON_KEY` | aynı sayfada `anon` `public` |
| `SUPABASE_SERVICE_ROLE_KEY` | aynı sayfada `service_role` `secret` |
| `TELEGRAM_BOT_TOKEN` | BotFather'dan |
| `TELEGRAM_CHAT_ID` | `getUpdates` JSON'undan |
| `RESEND_API_KEY` | resend.com API Keys |

## Komutlar (ben çalıştıracağım)

### 1. Supabase

```bash
# Bir kez senin browser oauth onayın gerek:
supabase login

# Sonrasını ben yapacağım:
cd /Users/archixusa/Desktop/bodrum-rezervasyon-paneli
supabase link --project-ref $SUPABASE_PROJECT_REF
supabase db push
supabase functions deploy notify-new-request

# Edge Function secrets:
supabase secrets set \
  TELEGRAM_BOT_TOKEN=$TELEGRAM_BOT_TOKEN \
  TELEGRAM_CHAT_ID=$TELEGRAM_CHAT_ID \
  RESEND_API_KEY=$RESEND_API_KEY \
  FROM_EMAIL=onboarding@resend.dev \
  ADMIN_EMAIL=info@bodrumapartkiralama.com \
  PANEL_URL=https://bodrum-rezervasyon-paneli.vercel.app

# Database Webhook (Edge Function tetikleyici) için Supabase Management API:
curl -X POST "https://api.supabase.com/v1/projects/$SUPABASE_PROJECT_REF/database/webhooks" \
  -H "Authorization: Bearer $SUPABASE_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "notify-on-new-request",
    "table_name": "reservation_requests",
    "events": ["INSERT"],
    "type": "supabase_function",
    "function_name": "notify-new-request"
  }'

# Test insert (bildirim gelmeli):
supabase db query "insert into reservation_requests (source_site, guest_name, guest_phone) values ('test', 'Setup Test', '+90 538 512 40 88');"
```

### 2. 4 site Vercel env'leri (CLI ile)

```bash
for repo in bodrumapartkiralama-com bodrumapartvilla-com bodruminsaatadilat bodrumacilsu; do
  cd /Users/archixusa/Desktop/site-integrations/$repo
  vercel link --yes --project $repo 2>/dev/null || true
  printf "%s" "$SUPABASE_URL"      | vercel env add NEXT_PUBLIC_SUPABASE_URL production preview
  printf "%s" "$SUPABASE_ANON_KEY" | vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production preview
done
```

### 4. Admin paneli (Vercel — dashboard'tan import edilmiş olsun)

Senin dashboard'tan import etmen lazım çünkü monorepo sub-directory ayarı CLI'da kararsız:

🔗 https://vercel.com/new → bodrum-rezervasyon-paneli → Root Directory: `admin-panel` → Deploy

Sonra:

```bash
cd /Users/archixusa/Desktop/bodrum-rezervasyon-paneli/admin-panel
vercel link --yes  # mevcut projeye bağlanır
printf "%s" "$SUPABASE_URL"      | vercel env add NEXT_PUBLIC_SUPABASE_URL production preview development
printf "%s" "$SUPABASE_ANON_KEY" | vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production preview development

```
